## 1. What Lumen Is

Lumen is a personal, multi-agent learning companion. Every capability — calendar,
email/communication, portfolio, a teaching-assistant bridge (Shiksha), Microsoft
Graph, plus a set of tool agents — is exposed as an addressable **A2A
(agent-to-agent) agent** with its own discovery card and JSON-RPC endpoint. Every
signed-in user *also* gets a personal Lumen that is itself an addressable peer
agent, so Lumens can discover and message each other.

The product surface is a single-page React app (chat-first, with voice, widgets,
and feature pages) backed by a FastAPI service.

## 2. System Architecture

### 2.1 High-level flow

```
User / Browser
      │
      ▼
FastAPI chat routes  ──►  Interaction Manager (app/agents/interaction_manager.py)
      │                         │  LLM-primary routing (classify → dispatch)
      │                         ▼
      │                   Shared A2A Client (app/agents/a2a_client.py)
      │                         │  JSON-RPC  tasks/send
      │           ┌─────────────┴───────────────┐
      ▼           ▼                             ▼
 Internal A2A agents              Peer Lumen A2A endpoint
 POST /a2a/{agent_id}             POST /a2a/lumen/{user_id}
   calendar, communication,         (message / schedule_meeting /
   portfolio, shiksha, graph,        info_request / remind)
   + external registered agents
      │
      ▼
 Cosmos DB  (disk / in-memory fallback)
```

### 2.2 Backend (FastAPI, Python 3.11)

Mounted routers (see [app/main.py](app/main.py)):

| Prefix | Router | Purpose |
| --- | --- | --- |
| `/auth` | auth | Entra + Google + GitHub identity, JWT issuance |
| `/chat` | chat | Primary chat ingress |
| `/lumen` | lumen_api | Lumen state, usage, widgets, email/comm proxies |
| `/lumen` | lumen_social | Peer discovery, LITP cards, peer messaging |
| `/agents` | registry | Agent cards, directory, external registration |
| `/a2a/...` | a2a | JSON-RPC endpoint for internal + external agents |
| `/a2a/lumen/...` | lumen_a2a | Each user's Lumen as a peer agent |
| `/portfolio` | portfolio | GitHub OAuth + portfolio artifact commits |
| `/coding-ta` | coding_ta | Coding TA generation + artifact listing |
| `/shiksha` | shiksha | Teaching-assistant bridge |
| `/notion`, `/google` | notion, google | Integration OAuth callbacks/proxies |

Key subsystems:

- **A2A protocol layer** — `app/protocols/` (`models.py` card schema, `a2a.py`
  JSON-RPC dispatch, `lumen_a2a.py` peer agents). Cards are served at
  `/.well-known/agent-card.json`, `/agents/{slug}/agent-card.json`,
  `/a2a/lumen/{user_id}/agent-card.json`, and aggregated at `/agents/directory`.
- **Orchestration** — `app/orchestrator/registry.py` holds the internal routing
  table and external-agent registration (identity = endpoint URL, slug = last
  path segment).
- **Agents** — `app/agents/`: calendar, communication, portfolio, shiksha, graph
  (Microsoft Graph mail), gmail, gcalendar, gdrive, notion, arxiv, wolfram,
  coding_ta, plus the interaction_manager (router) and llm_router.
- **Lumen core** — `app/lumen/`: identity/state (`core.py`), chat agent
  (`agent.py`), model routing, **pricing**, **token_tracker**, security, widgets,
  UX presets.
- **Persistence** — `app/db/cosmos.py`. Cosmos containers: `lumens`,
  `chat_threads`, `graph_tokens`, `peer_messages`, `agents`. Falls back to
  disk/in-memory when Cosmos isn't configured (local/demo).

### 2.3 Frontend (React + Vite + Tailwind)

`frontend/src/` — chat shell (`LumenChat`, `Shell`, `IconRail`, `WidgetZone`,
voice panels), and feature pages: Dashboard, Portfolio, CodingTA, Calendar,
Peers, Agents, **Usage**, TAPanel, Privacy, plus OAuth callback pages
(GitHub/Google/Notion/Entra/Outlook). Built into `app/static/` and served by the
backend; SPA routes are an **explicit allow-list** in `app/main.py`.

### 2.4 Models & infra

- **LLMs:** Azure OpenAI `gpt-5.4` (premium) and `gpt-54-mini` (cheap), plus
  Azure AI Foundry for the router's mini calls. All AOAI auth via
  managed-identity bearer-token provider.
- **Deploy:** Azure App Service `lumen-demo` (RG `nexus-rg`), Linux `PYTHON|3.11`,
  Oryx server-side build. Explicit zip packaging via `scripts/make_deploy_zip.py`
  + `deploy.ps1`.
- **Stack:** agent-framework-{core,openai,ag-ui,a2a,orchestrations}, FastAPI,
  uvicorn/gunicorn, azure-identity, azure-cosmos, PyGithub, tiktoken, httpx.

## 3. Recently Completed Work (early June 2026)

### 3.1 Usage & cost visibility — `/usage` (2026-06-08 → 06-09) ✦ newest

A token-usage and cost dashboard, built **advisory-only by design** — it surfaces
where spend goes and how to reduce it, but never auto-switches models or stops
agents.

- **Token tracker** ([app/lumen/token_tracker.py](app/lumen/token_tracker.py)):
  per-user session counters (in-memory) + persisted lifetime/daily totals on the
  Lumen doc, split by `source` (which agent), with per-call latency.
- **Pricing & recommendations** ([app/lumen/pricing.py](app/lumen/pricing.py)):
  maps each source → its predominant model, estimates USD cost, and produces
  model-swap recommendations with estimated savings. Central insight surfaced:
  `lumen-chat` (premium gpt-5.4) is the dominant cost driver despite low token
  volume — the lever is routing simple turns to gpt-54-mini.
- **Cost-spike detection:** per-call spike alerts (env-tunable `LUMEN_SPIKE_*`
  thresholds) surfaced via the notifications bell + the Usage page.
- **UI:** [frontend/src/pages/Usage.jsx](frontend/src/pages/Usage.jsx),
  [NotificationsBell.jsx](frontend/src/components/NotificationsBell.jsx).

### 3.2 Coding TA feature (2026-06-03)

A demo subject agent. Chat "coding ta" → in-app redirect to `/coding-ta`. The
page lets a student request code / quiz / notes / exercise; the LLM
([app/agents/coding_ta.py](app/agents/coding_ta.py), AOAI via managed identity)
generates it and it is **silently auto-committed** to the linked GitHub portfolio
(`coding-ta/<type>/<title-slug>.<ext>`, grouped by type, named by title).

- Fixed a base64 double-encoding bug across all three portfolio write paths
  (PyGithub base64-encodes itself — must pass raw content).
- Fixed two latent SPA gotchas: new client routes must be added to the main.py
  allow-list, and in-app `context_switch` must use the client router, not a popup
  redirect to the backend URL.

### 3.3 LLM-primary message routing (2026-06-01)

Rewrote `dispatch()` to be LLM-primary (keyword routing was mis-routing odd
phrasings). `classify_intent()` regex still runs first for LAUNCH deep-links and
fast-paths trivial chatter + PORTFOLIO; everything else goes to
`llm_classify_multi()` ([app/agents/llm_router.py](app/agents/llm_router.py)),
which returns multiple intents executed together.

- Added an **Azure OpenAI fallback** to the Foundry router call (Foundry
  `gpt-54-mini` was failing in prod with no fallback → every non-keyword message
  dropped to "Not sure what you mean"). Circuit breaker + 60s result cache.
- `gmail`/`communication` both route through `/a2a/communication`, which picks
  the provider by **account type**: Google users → Gmail; Entra users →
  Outlook/extension.

### 3.4 Auth / connect flows rework (2026-06-01 → 06-03)

- **GitHub:** moved to web OAuth (Authorization Code popup), no PAT anywhere.
- **Google:** login is identity-only; Gmail/Drive/Calendar access requested
  in-app on first use via an in-chat consent card — **Allow once** vs **Always
  allow** (once = online/no refresh token; always = offline + refresh token).
  One-time access is enforced Lumen-side, not via Google token expiry.
- Fixed the **AOAI auth gotcha** in the peer-Lumen autoreply (must use
  `azure_ad_token_provider`, not `api_key=<aad token>`; use
  `max_completion_tokens`) — peers had been repeating a canned fallback line.

## 4. What's Working Today

- Specialist agents expose A2A cards and accept JSON-RPC `tasks/send`.
- Shared A2A client can call any internal, external, or peer endpoint.
- Responses carry a dual artifact (plain text + structured JSON for UI cards).
- Every user has a Lumen identity (`lumen://tenant/user`) + peer A2A endpoint.
- Peer discovery, peer cards, and Lumen-to-Lumen messaging (deliver + persist).
- LLM-primary routing with keyword fallback and circuit breaker.
- GitHub web OAuth + Google once/always consent flows.
- Coding TA generation → silent auto-commit to GitHub portfolio.
- Usage/cost dashboard: per-agent cost share, cost spikes, swap recommendations.
- Learning progress stored on each user's Lumen doc.
